# Security policy

This repository is an independent community project.

It is not affiliated with, endorsed by, or supported by Control D.

This document only covers vulnerability reporting for this repository. For the
high-level security approach, architecture trade-offs, and awareness notes, see
`docs/ARCHITECTURE.md`.

## Reporting a vulnerability

Do not report security vulnerabilities in public issues or discussions.

Use GitHub's private vulnerability reporting for this repository:

- go to the Security tab in this repository
- use `Report a vulnerability`

If private reporting is unavailable for any reason, open a GitHub Discussion
and ask for a private contact path without posting vulnerability details.

## What to include

Please include:

- affected versions
- impact and attack scenario
- reproduction steps or proof of concept
- suggested mitigation if known

## Response expectations

- initial acknowledgement target: within 7 days
- triage and impact assessment target: within 14 days
- fix timeline depends on severity, complexity, and release timing

## Scope

This policy applies to the `controld-manager-ha` repository and its maintained
releases.
